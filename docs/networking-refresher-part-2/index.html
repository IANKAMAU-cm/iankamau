In this article I will be going through various concepts. Let’s first go through TCP vs UDP.

TCP and UDP are both transport protocols. They lie within layer 4 of the OSI model.

In order for a network application to communicate it has to choose how it wants to send its data. Either via TCP or UDP.

TCP

Transmission control protocol. It is a connection oriented protocol which
ensures quality of service by re-transmitting packets until all packets are received correctly.

It is the most widely chosen between the two because it can reliably send and receive data. TCP was designed to ensure delivery of all packets and minimize packet loss.

To start a reliable connection TCP uses a three way handshake:

SYN > SYN-ACK > ACK

If a network issue arises and some data segments are lost, TCP can recover them. It achieves reliable connections in three ways:

1.Acknowledgements

Using a reference from wikipedia an acknowledgement is is a signal that is passed between communicating processes, computers, or devices to signify acknowledgment, or receipt of message, as part of a communications protocol.

2.Sequencing

TCP will assign numbers to segments as they are sent.This way the receiving device can collect these segments, reorder them correctly and determine if any segments are missing.

3.Checksum

It is the result of a simple calculation run against the data. The checksum of the sender and receiver should match if data is transmitted successfully.

TCP is great for common tasks such as web browsing and file transfer.

UDP

User Datagram protocol. It is a connectionless protocol that is primarily used to establish low-latency and loss-tolerating connections between applications on the internet.

UDP speeds up transmissions by enabling the transfer of data before an agreement is provided by the receiving party.

It has none of the error handling and sequencing of TCP. It however provides checksums for data integrity. UDP does not also implement handshakes and thus exposes an application to any unreliability of the network.

UDP is considered unreliable because it has no guarantee of delivery, ordering and has no duplicate protection. It is suitable for purposes where error checking and correction are either not necessary or are performed in the application. e.g real-time and live systems like games, voice and video calls.

Port Numbers

Found in layer 4 (Transport layer) of the OSI model.

Computers and applications have ports which enable transmission of data. With reference to techtarget :

A port in networking is a software-defined number associated to a network protocol that receives or transmits communication for a specific service. A port in computer hardware is a jack or socket that peripheral hardware plugs into.

Let me give an example. Lets say you navigate to your favorite website.

i)The browser first sends a request either in http or https to the webserver.

The request has a destination port which is port 80 or 443 (80 is used by http and 443 by https).

It also has a source port; your browser opens a local unique port in your computer to enable connection to the server. Lets say our source port is 5000.

ii)The server receives the request, looks at the destination port (80) and passes it to the application (Apache or Nginx).

iii)The server will then send a response

The response has a destination port which is 5000.

It also has a source port which is port 80.

IP addresses gets the data to the computer and the port gets the data to the right application.

There are 65535 ports.

Common ports

21 — FTP (File Transfer Protocol)

22 — SSH (Secure Shell)

23 — Telnet

25 — SMTP (Simple Mail Transfer Protocol)

53 — DNS (Domain Name System)

110 — POP3 (Post Office Protocol)

80 — HTTP (Hyper Text Transfer Protocol)

443 — HTTPS (Hyper Text Transfer Protocol Secure)

139 and 445 — SMB (Server Message Block)

161 — SNMP (Simple Network Management Protocol)

There are many ports that you will encounter in the field used by different services.

IP addresses

IP addresses are in the network layer of the OSI model (Internet Protocol address). An IP address is a unique identifier assigned to each device connected to a computer network.

IP addresses may be represented in various formats: decimal, binary, hexadecimal and octal.

There are various types of IP addresses: Private IP addresses, Public IP addresses,static IP address and dynamic IP address.

Public and private IP addresses are indicative of the location of the network private being used inside a network while the public is used outside of a network.

Static and dynamic IP addresses indicate permanency. A static IP address is permanent while a dynamic IP address can change from time to time.

Public IP, Private IP and NAT

A public IP address is an address that can be accessed directly over the internet and is assigned to your network router by your internet service provider (ISP).

A private IP address is the address your network router assigns to your device. Each device within the same network is assigned a unique private IP address (sometimes called a private network address), this is how devices on the same internal network talk to each other.

Private IP addresses let devices connected to the same network communicate with one another without connecting to the entire internet.

NAT or Network Address Translation is a method of mapping devices in a private network to a single Public IP address. (Network Address Translation (NAT) conserves IP addresses by enabling private IP networks using unregistered IP addresses to go online).

There is also two versions of IP addresses:

i)IP V4 addresses

ii)IP V6 addresses

i)IP V4 addresses

A simple example is 10.1.76.99

An IP V4 address has four octets separated by a dot. Each octet contains 8 bits.Therefore an IP V4 address is 32 bits in length.

Each octet can contain any number from 0–255.

An IP V4 address is separated into two parts:

i)Network id

ii)Host

A subnet mask or CIDR notation is used to distinguish between the network id and host part and it always accompanies an IP address.

For example, an IPv4 address and its subnet mask may be 192.0.2.1 and 255.255.255.0, respectively. The CIDR notation for the same IP address and subnet is 192.0.2.1/24, because the first 24 bits of the IP address indicate the network and subnet.

IP addresses are also split into various classes:

Class A, class B, class C, class D and class E

Each class has a range of IP addresses.

Class A

IP addresses range from 1.0.0.0–126.255.255.255

It’s subnet is 255.0.0.0

It has 3 octets for host allocation and the total hosts are:16,777,214

Class B

IP addresses range from 128.0.0.0–191.255.255.255

It’s subnet is 255.255.0.0

It has 2 octets for host allocation and the total hosts are:65,534

Class C

IP addresses range from 192.0.0.0–223.255.255.255

It’s subnet is 255.255.255.0

It has 1 octet reserved for host allocation. Total hosts are: 254

ii)IP V6 addresses

Developed to combat the diminishing IP V4 addresses. An IP V6 address is 128 bits long.

It has 8 sections and each section is separated by a colon. A section is called a hextet. A hextet has 16 bits hence 16 x 8 =128 bits.

IP V6 addresses are hexadecimal meaning each character can either be between 0–9 and a-f.

Each hexadecimal character is made up of 4 binary bits and each bit can either be a 1 or 0.

MAC addresses

A MAC address stands for Media Access Control address. It lies in the data link layer of the OSI model. MAC addresses are made up of hexadecimal values.

MAC addresses act as the physical addresses for local communications. They show up in most IEEE 802 networks including: 802.3 (as well as Ethernet II), 802.5, 802.11 (Wi-Fi), 802.15 (Bluetooth), and the ITU-T G.hn standards.

It is a unique identifier that is used to identify individual electronic devices on a network. MAC addresses are physical addresses, cannot be changed and are unique.

The address is 48 bits long or 6 bytes.

A MAC address has two sections:

  1. OUI- Organizationally Unique Identifier. It identifies the vendor.
  2. NIC specific- Network Interface Controller specific.

There are 3 types of MAC addresses:

i)Udicast- a particular unique MAC address.

ii)Multicast- an address used for an application or protocol. It will be sent to all devices but only those with that application or protocol will listen to it.

Multicast addresses allow a source device to send a packet to a group of devices.

iii)Broadcast- It is sent to all devices within a network. A broadcast address looks like this: FF-FF-FF-FF-FF-FF
