  1. 21/tcp  ftp  vsFTPd 3.0.3

sudo searchsploit vsFTPd 3.0.3    -  remote denial of service

ftp-anon: Anonymous FTP login allowed

sudo ftp x.x.x.x    → ftp:ftp    I am able to login

Doing a listing reveals an image.

Get the file,  use steghide to reveal if there is data contained within the image.

sudo steghide info gum_room.jpg

sudo steghide extract -sf gum_room.jpg

There is a base64 encoded message, decode it with cyberchef. It is a shadow file that contains charlies encoded password.

Copy the entire hash from charlie and navigate to hashes.com. Paste the hash and search.

2. 22/tcp  ssh  OpenSSH 7.6p1

sudo searchsploit OpenSSH 7.6p1  → username enumeration

3. 80/tcp http Apache httpd 2.4.29

sudo searchsploit Apache httpd 2.4.29  → no results

Scan with whatweb, nikto and nuclei

  • look at wappalyzer
  • look for cookies
  • look at robots.txt and sitemap.xml
  • Inspect the source code
  • scan for directories and sub-directories

sudo wget http://10.10.114.193/key_rev_key

Navigating to x.x.x.x reveals a login page, since we cracked charlies password try logging in with the credentials.

I am redirected to a page where I can execute commands.

Use hack-tools or rev-shells to get a reverse shell. I use nc mkfifo. I get a shell. Spawn a tty shell.

Browsing through the file system I found an ssh key pair. They were in /home/charlie

Copy the private key, I create a new file in my machine (sudo touch charlie) and paste the contents of the private key.

Modify the file permissions of the private key. (sudo chmod 600 charlie)

Login via ssh: sudo ssh -i charlie charlie@10.10.140.177

Navigate to /home/charlie to get the user flag

Privilege escalation

Running sudo -l reveals that we can run vi as sudo

Use gtfobins to search if we can abuse the vi binary to escalate privileges.

I get a root shell

Navigate to /root   Do a directory listing to find a python file

Running the python file it asks for a key, we previously got the key earlier, enter it

Pwned
